# Lerning ogrotate

ในการตั้งค่า `logrotate` เพื่อเก็บล็อกไฟล์ (logs) ไว้เป็นเวลา 100 วันพอดีบน CentOS 7 คุณต้องตั้งค่าให้ระบบทำการหมุนเวียนล็อกเป็นรายวัน (**daily**) และเก็บสำรองไว้ **100** ชุด

เนื่องจากการเก็บล็อกไว้ถึง 100 วันอาจใช้พื้นที่ดิสก์เป็นจำนวนมาก จึงขอแนะนำอย่างยิ่งให้ทำการบีบอัด (compress) ไฟล์เหล่านั้นด้วย

นี่คือวิธีการตั้งค่าอย่างละเอียด:

### **ขั้นตอนที่ 1: สร้างหรือแก้ไขไฟล์การตั้งค่า**

บน CentOS 7 ไฟล์การตั้งค่า logrotate สำหรับแต่ละแอปพลิเคชันจะถูกเก็บไว้ที่ไดเรกทอรี `/etc/logrotate.d/`

ให้สร้างไฟล์ใหม่สำหรับเซอร์วิสของคุณ (หรือแก้ไขไฟล์ที่มีอยู่) ตัวอย่างเช่น หากคุณกำลังตั้งค่าล็อกสำหรับแอปพลิเคชันของคุณเอง:

```bash
sudo vi /etc/logrotate.d/myapp

```

### **ขั้นตอนที่ 2: เพิ่มการตั้งค่าแบบ 100 วัน**

คัดลอกและวางการตั้งค่าด้านล่างนี้ลงในไฟล์ โดยอย่าลืมปรับเปลี่ยน path ของล็อกไฟล์ (`/var/log/myapp/*.log`) ให้ตรงกับตำแหน่งจริงที่แอปพลิเคชันของคุณเก็บล็อกไว้:

```text
/var/log/myapp/*.log {
    daily
    rotate 100
    compress
    delaycompress
    missingok
    notifempty
    create 0640 root root
    dateext
}

```

### **ขั้นตอนที่ 3: ทำความเข้าใจคำสั่งต่างๆ**

นี่คือคำอธิบายของแต่ละพารามิเตอร์ เพื่อให้คุณนำไปปรับแต่งเพิ่มเติมได้ตามต้องการ:

* **`daily`**: ทำการหมุนเวียน (rotate) ล็อกทุกๆ วัน
* **`rotate 100`**: เก็บไฟล์ล็อกที่ rotate แล้วย้อนหลังไว้ 100 ไฟล์ก่อนที่จะลบไฟล์ที่เก่าที่สุดทิ้ง (เมื่อใช้คู่กับ `daily` จะเท่ากับ 100 วันพอดี)
* **`compress`**: บีบอัดไฟล์ล็อกเวอร์ชันเก่าด้วย gzip เพื่อประหยัดพื้นที่ดิสก์
* **`delaycompress`**: เลื่อนการบีบอัดไฟล์ล็อกของรอบก่อนหน้าไปทำในรอบการหมุนเวียนถัดไป มีประโยชน์ในกรณีที่โปรแกรมของคุณไม่สามารถปิดไฟล์ล็อกได้ทันที และอาจยังมีการเขียนข้อมูลลงในไฟล์เก่าอยู่ชั่วขณะหนึ่ง
* **`missingok`**: หากหาไฟล์ล็อกไม่พบ logrotate จะข้ามไปทำไฟล์ถัดไปโดยไม่แสดงข้อความแจ้งเตือน error
* **`notifempty`**: จะไม่ rotate ล็อกหากไฟล์นั้นว่างเปล่า (ไม่มีข้อมูล)
* **`create 0640 root root`**: สร้างไฟล์ล็อกอันใหม่ที่ว่างเปล่าขึ้นมาทันทีหลังจากการ rotate โดยใช้สิทธิ์การเข้าถึง (`0640`) และกำหนดเจ้าของ (`root root`) ตามที่ระบุ คุณควรปรับแก้ส่วนนี้ให้ตรงกับ user/group ที่แอปพลิเคชันของคุณใช้งานจริง
* **`dateext`**: ต่อท้ายชื่อไฟล์ที่เก็บถาวรด้วยวันที่แทนการใช้ตัวเลข (เช่น ต่อท้ายด้วย YYYYMMDD) วิธีนี้จะช่วยให้การค้นหาและจัดเรียงไฟล์ล็อกตลอดช่วงเวลา 100 วันทำได้ง่ายขึ้นมาก

### **ขั้นตอนที่ 4: ทดสอบการตั้งค่า (Dry Run)**

ก่อนที่จะรอให้ระบบอัตโนมัติ (cron job) ทำงาน คุณสามารถทดสอบการตั้งค่าใหม่ของคุณได้อย่างปลอดภัยเพื่อให้แน่ใจว่าไม่มีข้อผิดพลาด (Syntax errors) โดยใช้แฟล็ก `-d` (debug) ซึ่งจะเป็นการจำลองการทำงานโดยไม่มีการแก้ไขหรือลบไฟล์จริง:

```bash
sudo logrotate -d /etc/logrotate.d/myapp

```

ลองอ่านผลลัพธ์ที่แสดงบนหน้าจอเพื่อตรวจสอบว่า logrotate อ่านไฟล์ของคุณได้อย่างถูกต้อง และเข้าใจว่าต้องเก็บไฟล์ไว้ 100 ชุดหรือไม่

*(หมายเหตุ: Logrotate ต้องอาศัย cron ในการทำงานทุกวัน บน CentOS 7 ตัว cron job รายวันที่ทำหน้าที่เรียกใช้ logrotate จะอยู่ที่ `/etc/cron.daily/logrotate` ซึ่งโดยปกติแล้วระบบจะตั้งเวลาให้ทำงานในช่วงเวลาประมาณ 03:00 น.)*
